# 食譜收藏系統 - PRD (產品需求文件)

## 1. 專案概述
### 背景與動機
平常看到許多想做的料理或食譜，常常散落在不同的網頁、截圖或備忘錄中。為了更有效率地集中管理、快速尋找與應用這些食譜，因此需要打造一個專屬的「食譜收藏系統」。

### 目標用戶
- 系統擁有者：我自己（個人使用的專屬平台）。

### 核心價值主張
提供一個簡單、直覺的平台，讓我能夠輕鬆新增、修改、歸納自己的食譜庫，並且能根據手邊現有的食材，快速找到合適的料理靈感。

---

## 2. 功能需求

以下列出系統的 5 大核心功能與對應的使用者故事（User Stories）：

| 功能 | 使用者故事 (User Story) |
| :--- | :--- |
| **新增食譜** | 作為**我自己**，我希望**能新增食譜（包含名稱、食材、步驟等資訊）**，以便**將想嘗試的料理配方永久保存**。 |
| **搜尋食譜** | 作為**我自己**，我希望**能透過關鍵字搜尋食譜名稱**，以便**在眾多收藏中快速找到我想做的特定料理**。 |
| **根據食材推薦食譜** | 作為**我自己**，我希望**能輸入手上現有的食材，並讓系統推薦相關食譜**，以便**解決不知如何消耗剩餘食材的困擾**。 |
| **修改食譜** | 作為**我自己**，我希望**能編輯已建立的食譜內容（例如調整份量、修正做法）**，以便**隨時記錄與更新每次實作後的改良配方**。 |
| **刪除食譜** | 作為**我自己**，我希望**能刪除不再需要或測試不滿意的食譜**，以便**保持食譜庫的整潔與精煉**。 |

## 3. 非功能需求

### 技術限制
- **後端框架**：Flask (Python)
- **模板引擎**：Jinja2
- **資料庫中介**：SQLite (輕量級，非常適合單人使用情境)
- **前端技術**：HTML, Vanilla CSS (或視需求搭配順手的 CSS 工具，並著重美觀設計), Vanilla JavaScript

### 效能與安全考量
- **效能**：系統應保持極致輕量，食譜列表與搜尋結果應在 1 秒內載入完成。如果未來加入圖片上傳功能，需留意圖片壓縮與大小限制。
- **安全**：雖然主要為個人使用，但仍建議加入基礎的安全機制（例如簡單的登入驗證、防範基本的 SQL Injection 和 XSS 攻擊），並確保 `.sqlite` 檔案的存放位置安全，防止未經授權的存取或竄改。

## 4. MVP 範圍 (Minimum Viable Product)

### Must Have (必須擁有，第一階段核心體驗)
- 【Create】新增食譜（名稱、食材清單、製作步驟）。
- 【Read】瀏覽所有食譜列表及食譜詳細內容。
- 【Update】修改食譜內容。
- 【Delete】刪除食譜。
- 基礎關鍵字搜尋功能。

### Should Have (應該擁有，大幅提升使用價值)
- 根據食材關鍵字模糊比對，推薦符合的食譜。
- 簡易的食譜圖片上傳與展示功能。

### Nice to Have (最好擁有，後續迭代優化)
- 食譜分類標籤功能（如：主食、甜點、早餐、15分鐘快手菜）。
- 一鍵將食譜中的食材轉換為待買購物清單。
- 黑暗模式 (Dark Mode) UI 支援。
